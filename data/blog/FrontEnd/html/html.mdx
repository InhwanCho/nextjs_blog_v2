## form 태그

```html
<form action="/post" method="POST">
  <h3>제목</h3>
  <input value="<%=post.title%>" type="text" style="display: block" name="title" />
  <input value="<%=post.content%>" type="text" style="display: block" name="content" />
  <button type="submit">수정</button>
</form>
```

주로 사용되는 속성 값으로는 required, readonly(읽기 전용 필드), disabled, placeholder가 있습니다.

form 태그의 action기능을 통해 method로 http 통신 방법을 선택 후 action의 url로 요청을 보낼 수 있습니다.

input 태그의
value는 미리 작성되있는 value를 설정.(글 수정 시)
placeholder는 미리 작성되있는 텍스트를 투명하게 설정.(예를 들어, 텍스트를 입력해주세요 같은 문구)
type은 text, password 등을 설정.
name은 해당 form 태그로 요청이 갈때 JSON의 key값.

button 태그의 submit타입은 action이 작동되는 트리거 입니다.

> 단, server를 경유하지 않는 미니 프로젝트의 경우 action 기능을 굳이 사용할 필요가 없습니다.

## input 태그 & label 태그

input 태그는 `type 속성`을 통해 다양한 방법으로 데이터를 받을 수 있습니다.
대표적 type에는 text, textarea, password, email, date, checkbox 이 있습니다.

label 태그는 입력받는 필드를 설명할 때 사용합니다.
참고의 label 태그의 `htmlfor(구버전)와 for(최신버전)`는 동일합니다

```javascript
//예시 1. label과 input을 동일한 위치에 위치시킬 시 for와 id를 같은 값을 할당 해야합니다.
<label for='같은이름'>password : </label>
<input type='password' name='' value='' id='같은이름'/>

//예시 2. label의 하위태그에 input을 위치
<label>
  password :
  <input type='password' name='' value=''/>
</label>

------------------------------------------------------------------------
// input값을 그룹화 시키고 싶을 때 
// name값을 동일한 값을 할당.
<label>
  <input type="radio" value={title} name={name}/>
  <span>값 1</span>
</label>
<label>
  <input type="radio" value={title} name={name}/>
  <span>값 1</span>
</label>
------------------------------------------------------------------------
// fieldset, legend를 사용하면 그룹화하여 정보를 보낼 때 편리.
<fieldset>
  <legend>사용자 정보</legend>
  <ul>
    <li>
      <label for='id11'>id : </label>
      <input type='text' id='id11'/>
    </li>
    <li>
      <label for='password11'>password : </label>
      <input type='password' id='password11'/>
    </li>
  </ul>
</fieldset>
```
