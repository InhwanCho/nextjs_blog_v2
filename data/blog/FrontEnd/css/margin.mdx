CSS에서 마진 겹침(margin collapsing)은 많은 개발자들이 혼란스러워하는 주제 중 하나입니다.

이 현상은 서로 인접한 블록 요소들의 `상단 또는 하단 마진이 병합`되어 단일 마진처럼 보이는 것을 말합니다.

이 글에서는 이러한 현상을 이해하고 이를 해결하기 위한 Block Formatting Context의 역할에 대해 논의해보겠습니다.

## 마진 겹침 현상

마진 겹침 현상은 주로 세 가지 경우에 발생합니다.

1. 인접한 형제 요소들
2. 부모 요소와 첫 번째 또는 마지막 자식 요소
3. 빈 블록 요소

이러한 사례를 좀 더 구체적으로 살펴보겠습니다.

### 인접한 형제 요소들

두 형제 요소가 가까이 위치할 때, 그들의 마진은 한쪽의 크기로 병합됩니다. 예를 들어, 두 요소 각각의 상단 마진이 20px, 하단 마진이 15px이라면, 겹친 후의 마진은 20px이 됩니다.

```html:html
<div class="element1">첫 번째 요소</div>
<div class="element2">두 번째 요소</div>
```

```css:styles.css
.element1 {
  margin-bottom: 20px;
}

.element2 {
  margin-top: 15px;
}
```

### 부모 요소와 첫 번째 또는 마지막 자식 요소

부모 요소의 상단 마진과 첫 번째 자식 요소의 상단 마진, 또는 부모 요소의 하단 마진과 마지막 자식 요소의 하단 마진이 겹칠 때 마진 겹침 현상이 발생합니다.

### 빈 블록 요소

내용이 없는 블록 요소의 상단과 하단 마진이 겹치면, 두 마진은 하나로 합쳐집니다. 이 경우, 블록의 실제 높이는 0으로 계산됩니다.

## Block Formatting Context(BFC)

Block Formatting Context는 이러한 마진 겹침 현상을 해결할 수 있는 방법 중 하나입니다. 

BFC는 웹 페이지상에서 독립적인 렌더링 영역을 만들고, 그 안에 있는 요소들은 외부의 영향을 받지 않고 그들만의 레이아웃 규칙을 가집니다.

BFC를 생성하는 여렌 방법이 있지만, 가장 흔한 방법은 `overflow` 속성을 사용하는 것입니다.

```css:styles.css
.container {
  overflow: hidden;
}
```

이 방법을 사용하면 `.container` 내의 요소들은 새로운 BFC를 형성하고, 이에 따라 내부의 마진 겹침 현상을 겪지 않게 됩니다.

## 결론

CSS에서 마진 겹침 현상은 레이아웃을 설계할 때 고려해야 할 중요한 요소입니다. 

이 현상을 이해하고 적절하게 대응하기 위해서는 Block Formatting Context와 그것을 생성하는 방법을 알아야 합니다. 

위에서 설명한 예시와 설명을 통해 마진 겹침 현상을 이해하고, 더욱 효율적인 웹 페이지 레이아웃을 설계하는 데 도움이 되기를 바랍니다.
